# Kharon

When you just need to recheck your tasks at certain intervals of times.

Simpliest way to organize state reconciliation.

## Structure

Kharon - the actor, that recheks statuses and actualizes statuses of the tasks. You need only one kharon to rule them all.

Lymbo - container, where tasks belongs, while their are validating.

Harold - optional notificator, that executes callback when task is Done or Failed.
